Layout rules

Drop labels when a field is recoverable from its content: amount + currency pill, date with cal icon, dropdown with selected value, tags row with # glyph, comment with placeholder.
Use the connector for any parent → child relationship (group → category, account → sub-account, project → task). Indent 30px, vertical line at x=18, L-elbow at the bottom.
Replace state words with icon dividers. eye + solid line = active; eye-off + dashed line = inactive. Same recipe for any active/inactive, on/off, published/draft pair.
Collapse trailing actions when a picker opens (animate max-width to 0, opacity 0). Keep them visible in manage mode so the ✕ stays reachable.
Use --font-num for any value the user reads as a number/code: amounts, dates, ISO currency, date ranges, IDs.
Don't add labels back just because a screen feels empty — the field IS the label. If meaning isn't obvious, swap in a glyph (cal, #, search).
Don't use text buttons for + New / Manage / Close / Edit / Hide. Always icons.
Don't use solid bright accent as a section background — accent is reserved for primary action (Save), the currency pill, and selected/focus rings.

What's the same as v0.7.0

The dark navy palette, blue Save / red Scan split, header brand + version, Russian content, multi-currency model. Nothing about the data model changes.

What changes

  • Labels removed on Amount, Date, Currency, Group, Category, Event, Tags, Comment.
  • Group + Category become one indented unit with a connector instead of two stacked sections.
  • + New / Manage / Close become icon buttons (plus / cog / x).
  • Active / Inactive word labels become eye / eye-off icon dividers with solid/dashed rules.
  • Picker vs Manage are now distinct: clicking the field opens a simple selectable list; clicking the cog opens the edit/visibility manage list. Trailing actions collapse for the picker, stay visible for manage.
  • Hero amount moves from labeled field to a 64px hero with the currency pill clickable inline.
